对，这个方向需要修正：**NekoCalc 不应该只是“科学计算器 + 少量工程工具”**，而应该变成一个面向个人用户的 **全能计算工具箱**。

新的定位应该是：

```text
NekoCalc = 科学计算器 + 方程工具 + 电路工具 + 机械工具 + 财务工具 + 单位换算 + 图形分析 + 个人计算笔记
```

底部导航仍然保持简单：

```text
计算 ｜ 工具 ｜ 图形 ｜ 笔记
```

但 **“工具”页面要升级成超级工具中心**。

# 4. 工具详情页统一交互

所有工具都必须统一，不要每个工具一个风格。

```text
┌────────────────────────────┐
│ 电阻分压              ☆    │
│ Vin, R1, R2 → Vout         │
│                            │
│ 输入                       │
│ Vin      [ 3.3        V  ] │
│ R1       [ 10         kΩ ] │
│ R2       [ 20         kΩ ] │
│ Load     [ 可选       kΩ ] │
│                            │
│ 主结果                     │
│ Vout = 2.200 V             │
│                            │
│ 详细结果                   │
│ Idiv = 0.110 mA            │
│ Ptotal = 0.363 mW          │
│ 误差范围 = 2.156~2.244 V   │
│                            │
│ 公式                       │
│ Vout = Vin × R2/(R1+R2)    │
│                            │
│ 复制   保存   重置   笔记   │
└────────────────────────────┘
```

核心交互原则：

```text
输入时实时计算
单位自动识别
主结果大字显示
详细结果折叠显示
公式说明可以展开
支持收藏工具
支持保存到笔记
```

# 5. 工具系统要做成可扩展架构

不要为每个工具硬写一个完全独立页面。建议设计统一的 `ToolDefinition`。

```dart
class ToolDefinition {
  final String id;
  final String title;
  final String description;
  final ToolCategory category;
  final List<ToolInputDefinition> inputs;
  final List<ToolResultDefinition> results;
  final String formula;
  final String explanation;
}
```

输入定义：

```dart
class ToolInputDefinition {
  final String key;
  final String label;
  final String unit;
  final double? defaultValue;
  final bool optional;
}
```

结果定义：

```dart
class ToolResultDefinition {
  final String key;
  final String label;
  final String unit;
  final bool primary;
}
```

这样后面新增工具时，不需要重写整个页面，只要新增：

```text
工具定义
计算逻辑
测试用例
```

# 8. 结论

你说得对，之前工具页太单一了。

新版 NekoCalc 应该升级为：

```text
计算页：快速科学计算
工具页：全能工具中心
图形页：函数绘图与分析
笔记页：历史、收藏、公式、自定义工具
```

其中 **工具页** 是产品深度的关键，应该覆盖：

```text
方程数学
电路电子
机械工程
财务商业
物理科学
单位换算
编程数据
自定义公式
```

这样 NekoCalc 才像一个真正能长期使用的个人计算工作台，而不是一个普通科学计算器。
